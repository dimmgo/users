
## Налаштування проекту

### 1. Встановити залежності
```bash
composer install
```

### 2. Налаштувати підключення до бази даних
Відредагуйте файл `.env` (або `.env.local`):
```
DATABASE_URL="mysql://user:password@127.0.0.1:3306/db_name?serverVersion=8.0"
```

### 3. Створити базу даних
```bash
php bin/console doctrine:database:create
```

### 4. Виконати міграції
```bash
php bin/console doctrine:migrations:migrate
```

## Консольні команди

### Створення root-користувача
```bash
php bin/console app:create-root-user <логін> <пароль> <телефон>
```
**Приклад:**
```bash
php bin/console app:create-root-user admin Password123 +380971234567
```
Після виконання в консолі буде виведено `Bearer Token` для авторизації.

## Авторизація

Всі запити до `/v1/api/*` вимагають заголовка `Authorization`:
```
Authorization: Bearer <ваш_токен>
```

---

## Ролі та права доступу

| Роль | Права доступу |
|---|---|
| `ROLE_ROOT` | **Повний доступ.** GET, POST, PUT, PATCH, DELETE без обмежень по ідентифікаторам. Може змінювати ролі іншим користувачам. |
| `ROLE_USER` | **Обмежений доступ.** GET, PUT, PATCH тільки для **власного** профілю. POST та DELETE заборонено. Не може змінювати свою роль. |

## API Користувачів `/v1/api/users`

### 1. GET `/{id}` — Отримання даних
**Права:** `ROLE_ROOT` (будь-який користувач), `ROLE_USER` (тільки свій `id`).
**Відповідь `200 OK`:** Повертає `login` та `phone`.

### 2. POST `/` — Створення нового користувача (тільки для Admin)
**Права:** тільки `ROLE_ROOT`.
**Опис:** Створює нового користувача. Поле `role` є необов'язковим (за замовчуванням `ROLE_USER`).
**Тіло запиту:**
```json
{
  "login": "admin_user",
  "pass": "Password123",
  "phone": "+380971234567"
}
```
**Відповідь `201 Created`:** Повертає `id`, `login`, `phone` та **унікальний `token`** (показується лише один раз при створенні).

### 3. PUT `/{id}` — Повне оновлення профілю
**Права:** `ROLE_ROOT` (будь-який), `ROLE_USER` (тільки свій `id`).
**Опис:** Призначений для оновлення основних даних профілю.
**Тіло запиту:**
```json
{
  "login": "user_new_login",
  "pass": "NewPassword123",
  "phone": "+380979998877"
}
```

### 4. PATCH `/{id}` — Часткове оновлення
**Права:** `ROLE_ROOT` (будь-який), `ROLE_USER` (тільки свій `id`).
**Опис:** Дозволяє оновити лише окремі поля. Зміна поля `role` доступна **тільки для `ROLE_ROOT`**.
**Тіло запиту (всі поля необов'язкові):**
```json
{
  "login": "partial_login",
  "role": "ROLE_ROOT"
}
```
> **Увага:** Якщо `ROLE_USER` спробує передати поле `role`, він отримає `403 Forbidden`.

### 5. DELETE `/{id}` — Видалення
**Права:** тільки `ROLE_ROOT`.

## Валідація та помилки

| Поле | Правила валідації                                                                                    |
|---|------------------------------------------------------------------------------------------------------|
| `login` | Обов'язкове (POST/PUT), 8-50 символів, унікальне                                                     |
| `pass` | Обов'язкове (POST/PUT), 8-50 символів                                                                |
| `phone` | Обов'язкове (POST/PUT), 8-20 символів                                                                |
| `role` | Необов'язкове. Допустимі: `ROLE_USER`, `ROLE_ROOT`. За замовчуванням `ROLE_USER`. Тільки для адміна. |

**Коди відповідей:**
- `400 Bad Request` — Помилка валідації (неправильний формат полів).
- `401 Unauthorized` — Відсутній або неправильний Bearer Token.
- `403 Forbidden` — Недостатньо прав для виконання дії.
- `404 Not Found` — Користувача не знайдено.
- `409 Conflict` — Користувач з таким логіном вже існує.
